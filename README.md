## 为什么注册时验证码是空白的？

搬瓦工的注册页面用了 Google 的 reCAPTCHA 服务，本质上是一段 JavaScript，从 `www.google.com` 加载。Google 在国内被墙，这个脚本请求直接超时，验证框自然什么都显示不出来。

这不是网速问题，不是浏览器 bug，不是搬瓦工服务器问题。就是网络访问受限。

## 方法一：用浏览器插件重定向 reCAPTCHA 请求（推荐）

reCAPTCHA 有一个镜像地址：`recaptcha.net`，国内可以正常访问，内容和 `google.com` 的完全一致。只要把浏览器发出的请求地址从 `google.com/recaptcha/api.js` 换成 `recaptcha.net/recaptcha/api.js`，验证码就能正常加载。

实现这个替换，用浏览器插件就够了。

### Chrome / Edge 用户

1. 在扩展商店搜索并安装 **Header Editor**（这是个请求头修改插件）
2. 打开插件的配置页面，找到"导入和导出"
3. 在导入 URL 处填入下方 JSON 配置地址，点击下载并确认导入：
   `https://azurezeng.github.io/static/HE-GoogleRedirect.json`
4. 保存规则，刷新搬瓦工注册页面

验证框应该就能出来了。

### Firefox 用户

Firefox 的 Gooreplacer 插件可以直接解决这个问题：

1. Firefox 附加组件页面搜索安装 **Gooreplacer**
2. 打开插件设置，添加一条重定向规则：
   - 匹配类型：URL
   - 匹配规则：`https://www.google.com/recaptcha/api.js`
   - 重定向到：`https://recaptcha.net/recaptcha/api.js`
3. 保存，刷新注册页面

手机版 Firefox 也支持 Gooreplacer，移动端操作同理。

## 登录时收不到验证码邮件怎么办？

这是另一个完全不同的问题。搬瓦工从某个版本开始加入了"新设备验证"机制——换新电脑、新浏览器，或者清了 Cookie 重新登录，系统会要求用验证码确认身份。

验证码通过邮件发送，发件人是 `noreply@64clouds.com`，邮件主题是"Device verification"。

收不到的原因，九成是 QQ 邮箱的拦截。

### QQ 邮箱拦截处理

QQ 邮箱对来自海外服务器的邮件特别敏感，有时候直接拦截，连垃圾箱都不进。查看方式：

1. 进入 QQ 邮箱首页，找到右侧"自助查询"
2. 点击"收信查询"→"邮件拦截"
3. 看看是否有搬瓦工发来的邮件被拦截
4. 如果有，选择"取回"，并把 `noreply@64clouds.com` 加入白名单

加白名单的方法：进入 QQ 邮箱设置→"反垃圾"→"白名单"，把搬瓦工的发件地址加进去，后续就不会再被拦截。

### 换一个邮箱

说真的，如果没注册搬瓦工，建议直接用 Gmail 或者 163 邮箱注册。这两个邮箱收搬瓦工邮件基本没问题，不用额外配置任何东西。

已经注册了 QQ 邮箱的，可以在搬瓦工后台修改邮箱地址：登录后点右上角"Edit Account Details"，找到"Email Address"字段，改成新邮箱，保存。

### 用备用验证码代替邮件验证

如果邮件实在搞不定，搬瓦工还提供 Backup codes（备用验证码）方案：

1. 登录搬瓦工账户，进入"Account Security"
2. 点击"Get my backup codes"
3. 系统会生成一组一次性验证码，每个只能用一次
4. 把这些码保存下来，下次换设备登录时用其中一个代替邮件验证码

这是最稳的兜底方案，适合邮件环境长期不稳定的情况。

### 在后台直接查看邮件内容

就算邮件没送达你的收件箱，搬瓦工后台有一个"Email History"功能，记录了系统发给你的所有邮件。登录账号之后，从右上角菜单找到"Email History"，点进去可以看到每封邮件的原文。密码重置链接、验证码、账单通知都在这里。

不过有一个前提：你得能登进去才能看 Email History。如果是因为验证码登不进去而需要密码重置，就没法用这个方法了——所以建议在账号状态正常的时候就先配好备用验证码。

## 常见问题解答

**Q：我不在国内，也遇到了验证码加载不出来的问题，怎么回事？**

A：少数情况下，某些代理或 VPN 出口 IP 也会触发 reCAPTCHA 的安全检测，导致验证框加载异常。建议关闭代理，用本地 IP 直接访问注册页面重试。

**Q：搬瓦工验证码邮件发件人是哪个地址？**

A：登录验证码邮件的发件地址是 `noreply@64clouds.com`，账单、续费等邮件发件地址是 `support@bandwagonhost.com`。把这两个都加入白名单会更稳。

**Q：登录时跳过了新设备验证，之后会不会再要求验证？**

A：搬瓦工会记住"受信任设备"，同一台设备、同一浏览器下登录不需要重复验证。但清了 Cookie、换了浏览器或者换设备，就会再次触发验证。

**Q：注册时填了 QQ 邮箱，现在想换邮箱怎么操作？**

A：登录搬瓦工后台，点右上角"Edit Account Details"，修改"Email Address"字段，填入新邮箱，保存即可。之后的登录和通知都会发到新邮箱。
